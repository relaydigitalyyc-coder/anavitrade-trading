# Anavitrade Production PRD — July 10, 2026

## Current State

The platform is 80% production-ready. What works:

- **Live signal pipeline**: Coinlegs scraper cron (`* * * * *`), 6-component scoring, confluence, SL/TP
- **Exchange integration**: Binance + Bitunix clients, per-user encrypted keys, trade-only verification
- **Execution engine**: Risk gating, dispatch with 3-retry, idempotency, kill switches
- **Consumer UI**: Exchange onboarding, dashboard, equity curve, zort-calibrated dark visual system
- **Shared lifecycle**: TradeIntent → ExecutionJob → OrderEvent → NavSnapshot (Aster + CEX)
- **Remote D1**: All migrations applied, `cex_connections` live, `trade_intents` schema ready

What's NOT ready for production:

| Gap | Severity | Owner |
|-----|----------|-------|
| No backtest with real outcomes | CRITICAL | Data |
| Signal quality unverified (all-time D1: avg profit 0.49%) | CRITICAL | Data |
| SMC validator built but untested against live signals | HIGH | Strategy |
| Fee engine schema-only (no 2-and-20 accrual) | HIGH | Platform |
| No portfolio-wide exposure cap | HIGH | Risk |
| No outcome validation (MaxProfit claims unverified) | HIGH | Data |
| No Sharpe/profit-factor/expectancy dashboards | MEDIUM | Analytics |
| Tier B with strong confluence not dispatched | MEDIUM | Strategy |
| Orphaned intents never retried | MEDIUM | Platform |
| No A/B cohort comparison (what if we traded ALL signals?) | MEDIUM | Research |

## Production Definition

Anavitrade is production-ready when:

1. **The algorithm is backtested** — 500+ historical signals with outcome-verified profit/loss against actual Binance klines, showing positive expectancy
2. **Risk is bounded** — per-user daily loss limit, portfolio exposure cap, and global kill switch all enforced in code (not just schema)
3. **Fees accrue automatically** — 2% management + 20% performance computed from NAV snapshots, high-water mark tracked, payment records created
4. **The dashboard is honest** — shows real verified metrics (Sharpe, profit factor, win rate, max drawdown), not hardcoded numbers
5. **Exchange connections validated** — at least one live testnet Binance trade with real testnet keys, from signal → intent → dispatch → fill → snapshot

## Workstreams (parallel-executable)

### Workstream A: Backtest & Signal Validation [DATA]

**Output**: 500+ outcome-verified signals, positive backtest result

**Task A1 — Historical Corpus**
- Feed 71 pages × 100 signals from coinlegs API (30-day window) into D1
- Result: ~7,000 signals scored, ~1,000+ buy signals with Tier A/B/C labels

**Task A2 — Outcome Validation**
- New `outcomeValidator` cron: for each signal with `outcomeValidated = false`, query Binance public klines for the signal's `marketName` + `period` + `maxProfitDuration` window
- Compute: did price actually hit the claimed `maxProfit`? What was the actual max drawdown?
- Store `outcomeValidated = true`, `actualMaxProfit`, `actualDrawdown` on `coinlegs_signals`

**Task A3 — Backtest Report**
- Query D1: for every Tier A/B/C signal with outcome data, compute expectancy, win rate, profit factor, Sharpe ratio
- Group by: tier, timeframe, indicator, market cap
- Output: `coinlegs_backtest_YYYY-MM-DD.json` with statistical significance and deploy/no-deploy recommendation

**Task A4 — Cohort Comparison**
- "What if we traded ALL buy signals?" vs "Tier A only" vs "Tier A + structural SMC gate"
- Show which filter provides the best risk-adjusted return

### Workstream B: Production Hardening [PLATFORM]

**Output**: Fee engine, exposure cap, orphan intent recovery, honest dashboards

**Task B1 — Fee Engine**
- `src/server/fee/engine.ts`: `crystallizeFees()` — for each active user: open fee period, compute NAV from snapshots, 2% mgmt, 20% perf above HWM, create `feePayments` rows
- Cron: run daily at 00:05 UTC
- Dashboard: show accrued fees YTD, next crystallization date

**Task B2 — Portfolio Exposure Cap**
- `decideExecution()`: before approving job, sum all open notional from `executionJobs` where status IN (`queued`,`submitted`)
- Reject if `(totalOpen + proposed) / equity > maxTotalExposurePct` (new column on `live_accounts`, default 25%)
- Add `maxTotalExposurePct` to settings UI

**Task B3 — Orphaned Intent Recovery**
- Cron (every 5 min): query `tradeIntents WHERE status = 'created' AND createdAt > NOW() - 6 hours`
- Re-run `createExecutionJobsForIntent` for each
- Update intent status to `dispatched` or `partially_dispatched`

**Task B4 — Dashboards**
- Replace hardcoded stats on HistoricalPerformance with live tRPC queries
- New `signals.performance` endpoint returning: totalSignals, tierDistribution, avgProfitByTier, winRate, profitFactor, sharpeRatio
- Show equity curve for demo from actual signal outcomes
- Add "Platform Health" panel: last scraper run, signals in DB, connected exchanges, fee accrued

**Task B5 — Production Deploy Checklist**
- Set Cloudflare secrets: `ENCRYPTION_KEY`, `JWT_SECRET` (both already done)
- Verify remote D1 has all migrations
- Binance testnet: connect testnet key, verify validate → balance → order placement → fill in testnet
- Enable cron triggers in production dashboard
- Document rollback procedure

### Workstream C: Strategy Refinement [STRATEGY]

**Output**: SMC gate validated, Tier B dispatch, coil score built on real data

**Task C1 — SMC Gate Validation**
- Apply `validateStructure()` to the full historical corpus
- Measure: what percentage of past Tier A signals would have been rejected? Of those rejected, what was their actual performance?
- Goal: prove the SMC gate improves win rate by ≥10 percentage points

**Task C2 — Tier B + Strong Confluence Dispatch**
- When a signal scores Tier B (40-64) AND `confluenceCount >= 4`, create intent with 50% position size
- Track separately in dispatch logs for performance analysis
- AB test: compare Tier-B-dispatched outcomes vs Tier-B-skipped outcomes

**Task C3 — Coil Score on Live Data**
- Port the simplified 5-component coil detector from ICR (range contraction, ATR contraction, BB squeeze, volume dry-up, higher lows)
- Compute on every live buy signal using the available coinlegs metadata
- Use as position-size modifier only (not gate) — high coil = full size, low coil = 75% size

### Workstream D: Analytics & Intelligence [RESEARCH]

**Output**: Live performance dashboards, research corpus, edge validation framework

**Task D1 — Performance Metrics Endpoint**
- `signals.performance`: live expectancy, Sharpe, profit factor, win rate by tier/timeframe/indicator
- `signals.cohortAnalysis`: A/B comparison of filter strategies
- Cached hourly, queries D1 directly

**Task D2 — Indicator Performance Matrix**
- For each of the 5 indicators (MACD, Stochastic, CCI, Ichimoku, Trend Reversal) × 7 timeframes:
  - Signal count, avg profit, win rate, best/worst pair
- Display on HistoricalPerformance page as a live heatmap

**Task D3 — Weekly Research Digest**
- Cron: Monday 09:00 UTC → query D1 for last 7 days → produce `research_digest_YYYY-WW.json`
- Contents: new signals, tier distribution, best/worst performers, new markets, any statistical anomalies
- Stored in D1 for trend analysis
