# Profit Engine Audit — July 10, 2026

Three independent audits converged. 7 critical, 8 high, 5 medium.

## CRITICAL (blocking)

### 1. MARKET Orders: Quantity Always "0" [3/3 agents]
`dispatch.ts:88-91` — `priceHint = parseFloat(null ?? "0")` for market orders. No live trade has ever executed.
→ Fix: add `limitPrice: String(sig.price)` to scraper TradeIntent insert. Use `quoteOrderQty` on Binance.

### 2. No Stop-Loss or Take-Profit [3/3 agents]
`coinlegs-scraper.ts:245` — `stopLossPrice: null, takeProfitPrice: null`. Every trade enters unprotected.
→ Fix: derive SL from signal metadata, TP from maxProfit.

### 3. Daily Loss Limit Never Enforced [2/3 agents]
`riskEngine.ts` reads maxDailyLossPct but never checks realized PnL.
→ Fix: query today's realized PnL from execution_jobs, deny if over threshold.

### 4. Confluence Counts Rows, Not Distinct Indicators [2/3 agents]
`coinlegs-scraper.ts:205-209` — counts same-market rows not unique indicator types.
→ Fix: count distinct indicatorName values per (market, period).

### 5. Permanent Dedup Blocks Re-Entries [2/3 agents]
`coinlegs-scraper.ts:187-189` — once (market, period, indicator) is stored, never fires again.
→ Fix: time-windowed dedup (24h) or per-position check.

### 6. Demo Simulator: Pyramiding Off by 100x [1/3 agents]
`db.ts:394` — `pyramidScalePct / 100` divides "0.50" by 100 → 0.005. All pyramided entries near-zero.
→ Fix: remove extra /100 division.

### 7. Demo Simulator: Only Winners Recorded [1/3 agents]
`db.ts:379` — `if (maxProfitPct <= 0) continue` skips all losing signals.
→ Fix: apply stop-loss simulation. Record both wins and losses.

## HIGH

### 8. Tier B + Strong Confluence Never Dispatched
Coinlegs-scraper.ts dispatches Tier A only. Tier B with ≥4 confluence is a genuine signal.
→ Fix: dispatch Tier B with ≥4 confluence at 50% size.

### 9. Failed Balance Check Falls Through to Job Creation
`dispatch.ts:81-93` catch block falls through, creates zero-quantity job.
→ Fix: skip connection on failure, don't create job.

### 10. Demo PnL Missing Leverage Multiplier
`db.ts:400-402` — undercounts leveraged returns.
→ Fix: add leverage to PnL formula.

### 11. No Total Exposure Cap
Per-trade 10% sizing with no cap on total open notional. 5 simultaneous signals = 50%.
→ Fix: check total open notional vs equity before approving.

### 12. Orphaned Intents Never Retried
Intents created when no connections active stay "created" forever.
→ Fix: cron re-scans pending intents.

### 13. No Retry on Transient Exchange Errors
Single fetch failure → permanent "error" status.
→ Fix: 3-retry exponential backoff.

### 14. Fee Model Schema-Only
fee_periods, fee_payments, highWaterMarkUsd exist but nothing computes them.
→ Fix: fee calculation engine in cron.

### 15. SL/TP Placement Failure Silently Swallowed
`binance.ts:107,113` — `.catch(() => undefined)` means unprotected position with no alert.
→ Fix: log error, update execution_jobs.

## MEDIUM

### 16. Bitunix getPositions Wrong Endpoint
`bitunix.ts:121` — `get_pending_positions` likely returns unfilled orders, not positions.

### 17. No Drawdown Protection
No peak-to-trough monitoring.

### 18. Stale Proxy Could Mask New Signals
Proxy Worker fetch has no detection window.

### 19. Single Exchange Source
Only Binance. Bybit and OKX have comparable liquidity.

### 20. No Backfill Mechanism
No way to pull historical signals for ongoing research.
