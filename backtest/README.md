# Backtest Dashboard — Separate Cloudflare Pages Deployment

This is a **standalone SPA** for visualizing backtest results from the Anavitrade ML pipeline. It is NOT a route in the main app. It deploys as a separate Cloudflare Pages project.

## Architecture

```
backtest/
  index.html                     ← Standalone entry point (React via CDN)
  dashboard.js                   ← All dashboard code (React + htm + lightweight-charts)
  backtest-dashboard-data.json   ← Backtest report data (co-located static JSON)
  wrangler.toml                  ← Cloudflare Pages config (separate project)
  README.md                      ← This file
```

## How It Works

- **React 18** is loaded from **esm.sh CDN** via importmap (no build step needed).
- **htm** provides JSX-like syntax without a transpiler.
- **lightweight-charts** is loaded from **unpkg** for the equity curve.
- **Tailwind CDN** provides utility classes. Custom CSS covers the glass-card UI.
- **Access gate**: The page requires `?key=anavitrade-founder-2026` query parameter to render.

## Deploy to Cloudflare Pages

### Option A: Via Wrangler CLI

```bash
cd backtest/

# Install Wrangler if needed
npm install -g wrangler

# Deploy static files
npx wrangler pages deploy . --project-name anavitrade-backtest-dashboard

# Set the founder key as an environment variable (optional — defaults are in the code)
npx wrangler pages secret put FOUNDER_KEY --project-name anavitrade-backtest-dashboard
```

### Option B: Via Cloudflare Dashboard

1. Go to Cloudflare Dashboard > Pages > Create a project.
2. Connect your Git repository.
3. Set:
   - **Build command**: `echo "static"`
   - **Build output directory**: `backtest`
   - **Root directory** (if supported): `/backtest`
4. Add environment variable (optional):
   - `FOUNDER_KEY = anavitrade-founder-2026`

## Updating Data

The dashboard reads `backtest-dashboard-data.json` from the same directory. To update with a new backtest report:

```bash
cp /home/ariel/anavitrade-trading/scripts/data/backtest/backtest_report_<timestamp>.json \
   /home/ariel/anavitrade-trading/backtest/backtest-dashboard-data.json
```

**Note**: The real data is generated by `scripts/ml/build-backtest-dashboard-json.py`. The included JSON file is a copy of the latest backtest report for initial setup — replace it with the properly formatted dashboard JSON when that pipeline is complete.

## Local Development

No build tools needed. Just open `index.html` in a browser with `?key=anavitrade-founder-2026`:

```bash
cd /home/ariel/anavitrade-trading/backtest

# Using Python's HTTP server (needed for fetch() to work)
python3 -m http.server 8080
# Then open: http://localhost:8080/?key=anavitrade-founder-2026

# Or using npx serve
npx serve . -p 8080
# Then open: http://localhost:8080/?key=anavitrade-founder-2026
```

## Security

- The access gate (`?key=`) is a simple client-side check. It prevents casual browsing but does **not** provide real security.
- For production, add a Cloudflare Access policy in front of the Pages deployment.
- The `FOUNDER_KEY` env var in Pages only documents the expected value — the actual check is hardcoded in `dashboard.js`. Change the `ALLOWED_KEYS` array if you change the key.

## Notes

- No files in the main `src/` directory or the main `wrangler.toml` are modified by this deployment.
- This dashboard is fully self-contained and can be deployed independently.
